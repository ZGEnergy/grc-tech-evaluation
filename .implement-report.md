# Implementation Report: PRD 03/02 - ACPF Reference Solution Extraction

## Summary

Implemented a Python module that produces the AC Power Flow (ACPF) reference
solution dataset for the FNM Annual S01 network. The module handles two mutually
exclusive paths based on D8 snapshot classification: solved-case extraction
(direct CSV read) and flat-start solver execution (MATPOWER runpf via Octave).
Outputs include per-bus VM/VA, per-branch P/Q flows, per-generator P/Q, and
a metadata JSON with system-level summary statistics.

## Design Decisions

- Column auto-detection supports both header-bearing CSVs (GridCal) and headerless
  CSVs (MATPOWER 13-column bus matrix format)
- Bus exclusion: inline IDE=4/VM=0 filtering plus optional D1 registry consumption
- System summary: generation from gen P/Q, load from bus PD/QD, losses from branch flows
- Precision: VM at 8 decimal places, VA at 6, P/Q at 4 (exceeds PRD minimums)
- Flat-start MATPOWER path uses two-stage Q-limit enforcement via Octave subprocess
- GridCal solver path raises NotImplementedError (documented as future work)
- Supplemental files (taps_acpf.csv, shunts_acpf.csv) written as header-only
  placeholders per OQ-P3-05
