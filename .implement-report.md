# Implementation Report: PRD-01.01 Dockerfile

## What Was Built

A single-stage Dockerfile at `.devcontainer/Dockerfile` that produces a container
image with three language runtimes and all six power-system tool dependency sets
pre-installed.

## Layer Summary

| Layer | Contents | Cache Stability |
|-------|----------|-----------------|
| 1 | Base image (`devcontainers/base:ubuntu-24.04`) | Very stable |
| 2 | System packages: Python 3.12, Octave 8.x, curl, unzip | Stable |
| 3 | Julia 1.10.7 from official tarball | Stable (pinned version) |
| 4 | uv via multi-stage COPY | Stable |
| 5 | `JULIA_CPU_TARGET=generic` env var | Stable |
| 6 | Full repo COPY to `/workspace` | Changes on any source edit |
| 7 | `uv sync` for gridcal, pandapower, pypsa | Rebuilds when lock files change |
| 8 | Julia `Pkg.instantiate()` + `Pkg.precompile()` for powermodels, powersimulations | Rebuilds when Project.toml/Manifest.toml change |
| 9 | MATPOWER 8.1 download via `setup.sh` | Rebuilds when setup.sh changes |

## Key Decisions

- **Option A (self-contained Dockerfile):** Python 3.12 installed via apt (Ubuntu 24.04
  ships it natively) and Julia 1.10.7 installed from the official tarball. This ensures
  all runtimes are available during the Docker build for dependency baking, rather than
  relying on devcontainer features which run after the build.
- **Julia version pinned to 1.10.7:** Latest patch release in the 1.10 series, set via
  a build ARG for easy updates.
- **Separate RUN per tool:** Each `uv sync` and Julia instantiate command is a separate
  RUN layer so that a failure in one tool does not invalidate cached layers for others.

## Implementation Report: Python Gate Tests

## PRD ID: 01, Phase: 02

## Summary

Implemented pytest-based gate test suites for all three Python tools (pypsa, pandapower, gridcal). Each tool now has a `tests/conftest.py` with a shared `data_dir` fixture and a `tests/test_gate.py` with three gate tests: import, parse, and solve.

## Files Created

- `evaluations/pypsa/tests/conftest.py` -- shared data_dir fixture
- `evaluations/pypsa/tests/test_gate.py` -- 3 gate tests (import, parse case39, DC power flow via lpf)
- `evaluations/pandapower/tests/conftest.py` -- shared data_dir fixture
- `evaluations/pandapower/tests/test_gate.py` -- 3 gate tests (import, parse case39, DC power flow via rundcpp)
- `evaluations/gridcal/tests/conftest.py` -- shared data_dir fixture

## Deviations from PRD

1. **Removed unused `import pytest`** from all three `test_gate.py` files. The PRD included `import pytest` at module level, but since no pytest markers or fixtures are used directly in the test file (the `data_dir` fixture comes from conftest.py), ruff flagged it as F401 (unused import). Removed to pass lint.

2. **Added `# noqa: F401`** to GridCal's `test_import` for `import VeraGridEngine as vge`. The import is the test itself (verifying the library can be imported), but the `vge` alias is not referenced in that method since the version is retrieved via `importlib.metadata`. The noqa comment suppresses the lint warning while preserving the PRD's intent of testing the import. Ruff format subsequently collapsed the `test_parse_case39` assertion onto a single line (within the 100-char limit).
