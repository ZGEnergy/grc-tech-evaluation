## What was built

Scalability benchmark line plot renderer for the chart pipeline with 3-tier graceful degradation:

1. **FULL_TIMING** — Line plots (Scatter, `mode="lines+markers"`) of solve time vs. network size, one trace per tool, log y-axis
2. **PASS_FAIL_ONLY** — Bar charts of pass counts per tool when timing data lacks solve times
3. **NO_DATA** — Returns empty list (nothing to render)

## Files created

- `report/scripts/renderers/__init__.py` — Package init
- `report/scripts/renderers/scalability.py` — Renderer implementation
- `report/scripts/tests/test_scalability_plots.py` — 16 tests

## Key details

- Registered as `"line"` renderer via `register_renderer("line", render_scalability_plots)`
- Uses `TOOL_COLORS` for trace colors and `TOOL_MARKERS` for marker symbols
- Legend positioned outside plot area (`x=1.02`)
- Chart IDs follow `line_scalability-{benchmark_type}` format
- All charts use `ChartType.LINE` and `data_source="test-results.json"`
