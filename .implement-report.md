# Implementation Report: PRD 06/02 — Chart-Manifest Cross-Reference Validator

## Summary

Created a bidirectional validation script ensuring consistency between chart-manifest.json,
chart image files on disk, and MDX source files embedding charts.

Forward validation checks that every MDX chart reference has a manifest entry and file on
disk. Reverse validation checks that every manifest entry is referenced by at least one MDX
file (orphaned = warning, missing file = error).

## Data Structures

## API

- `load_manifest(manifest_path)` — parses chart-manifest.json
- `scan_mdx_chart_references(docs_dir)` — finds chart image refs in .mdx files
- `validate_forward(references, manifest, static_dir)` — MDX -> manifest + disk
- `validate_reverse(manifest, references)` — manifest -> MDX (orphan warnings)
- `validate_chart_manifest(docs_dir, static_dir, manifest_path)` — full validation
- CLI: exit 0 (pass), exit 1 (errors), exit 2 (manifest missing/unparseable)
