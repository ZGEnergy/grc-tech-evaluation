# Implementation Report: PRD 03/04 - DCPF-vs-ACPF Characterization

## Summary

Implemented a Python module that compares the DCPF reference solution (D3) against
the ACPF reference solution (D2) to characterize DC power flow approximation quality.
The module loads pre-computed reference CSVs, joins on bus number and composite branch
key, computes per-element deviation metrics, identifies worst-case elements with
probable cause annotations, and writes machine-readable JSON and human-readable
markdown reports.

## Design Decisions

- Inner join with branch key normalization to (min, max, ckt) handles reversed
  direction between ACPF and DCPF references
- Near-zero flow exclusion at 1.0 MW threshold for percentage deviation metrics
- Cause annotation via rule-based heuristics in priority order: phase_shifter,
  high_reactance, heavy_loading, low_voltage, high_voltage, transformer_tap,
  slack_bus_vicinity, uncategorized
- stdlib only: all computations use math and statistics modules (no numpy/scipy)
- Percentile computation uses linear interpolation matching numpy default behavior
- Expected-range checks are informational (warnings only), not pass/fail gates
