# Implementation Report: PRD-01.01 Dockerfile

## What Was Built

A single-stage Dockerfile at `.devcontainer/Dockerfile` that produces a container
image with three language runtimes and all six power-system tool dependency sets
pre-installed.

## Layer Summary

| Layer | Contents | Cache Stability |
|-------|----------|-----------------|
| 1 | Base image (`devcontainers/base:ubuntu-24.04`) | Very stable |
| 2 | System packages: Python 3.12, Octave 8.x, curl, unzip | Stable |
| 3 | Julia 1.10.7 from official tarball | Stable (pinned version) |
| 4 | uv via multi-stage COPY | Stable |
| 5 | `JULIA_CPU_TARGET=generic` env var | Stable |
| 6 | Full repo COPY to `/workspace` | Changes on any source edit |
| 7 | `uv sync` for gridcal, pandapower, pypsa | Rebuilds when lock files change |
| 8 | Julia `Pkg.instantiate()` + `Pkg.precompile()` for powermodels, powersimulations | Rebuilds when Project.toml/Manifest.toml change |
| 9 | MATPOWER 8.1 download via `setup.sh` | Rebuilds when setup.sh changes |

## Key Decisions

- **Option A (self-contained Dockerfile):** Python 3.12 installed via apt (Ubuntu 24.04
  ships it natively) and Julia 1.10.7 installed from the official tarball. This ensures
  all runtimes are available during the Docker build for dependency baking, rather than
  relying on devcontainer features which run after the build.
- **Julia version pinned to 1.10.7:** Latest patch release in the 1.10 series, set via
  a build ARG for easy updates.
- **Separate RUN per tool:** Each `uv sync` and Julia instantiate command is a separate
  RUN layer so that a failure in one tool does not invalidate cached layers for others.

## Implementation Report: Python Gate Tests

## PRD ID: 01, Phase: 02

## Summary

Implemented pytest-based gate test suites for all three Python tools (pypsa, pandapower, gridcal). Each tool now has a `tests/conftest.py` with a shared `data_dir` fixture and a `tests/test_gate.py` with three gate tests: import, parse, and solve.

## Files Created

- `evaluations/pypsa/tests/test_gate.py` -- 3 gate tests (import, parse case39, DC power flow via lpf)
- `evaluations/pandapower/tests/conftest.py` -- shared data_dir fixture
- `evaluations/pandapower/tests/test_gate.py` -- 3 gate tests (import, parse case39, DC power flow via rundcpp)
- `evaluations/gridcal/tests/conftest.py` -- shared data_dir fixture
- `evaluations/gridcal/tests/test_gate.py` -- 3 gate tests (import, parse case39, DC power flow via Linear solver)

## Files Modified

- `evaluations/pypsa/pyproject.toml` -- added `[dependency-groups] dev = ["pytest"]` and `[tool.pytest.ini_options]`
- `evaluations/pandapower/pyproject.toml` -- same additions
- `evaluations/gridcal/pyproject.toml` -- same additions
- `evaluations/pypsa/uv.lock` -- updated by uv sync (pytest added)
- `evaluations/pandapower/uv.lock` -- updated by uv sync (pytest added)
- `evaluations/gridcal/uv.lock` -- updated by uv sync (pytest added)

## Deviations from PRD

1. **Removed unused `import pytest`** from all three `test_gate.py` files. The PRD included `import pytest` at module level, but since no pytest markers or fixtures are used directly in the test file (the `data_dir` fixture comes from conftest.py), ruff flagged it as F401 (unused import). Removed to pass lint.

2. **Added `# noqa: F401`** to GridCal's `test_import` for `import VeraGridEngine as vge`. The import is the test itself (verifying the library can be imported), but the `vge` alias is not referenced in that method since the version is retrieved via `importlib.metadata`. The noqa comment suppresses the lint warning while preserving the PRD's intent of testing the import. Ruff format subsequently collapsed the `test_parse_case39` assertion onto a single line (within the 100-char limit).

## Implementation Report: PRD 01/01 - FNM Directory Structure & Manifest

## Summary

Created the `data/fnm/` directory tree with manifest, gitignore, and manifest I/O module that all subsequent Phase 1 FNM deliverables build on.

## Files Created

| File | Purpose |
|------|---------|
| `data/fnm/README.md` | Top-level README documenting layout, NDA restrictions, FNM_PATH usage |
| `data/fnm/.gitignore` | Blocks data files (`*.raw`, `*.csv`, `*.parquet`, `*.m`, `intermediate/**`, `reference/**`), allows infrastructure files |
| `data/fnm/manifest.json` | Machine-readable manifest listing 1 PSS/E RAW + 7 supplemental CSVs with placeholder checksums |
| `data/fnm/__init__.py` | Package init |
| `data/fnm/intermediate/README.md` | Subdirectory README for parser output |
| `data/fnm/reference/README.md` | Subdirectory README for verification datasets |
| `data/fnm/docs/README.md` | Subdirectory README for reference documentation |
| `data/fnm/scripts/README.md` | Subdirectory README for Python modules |
| `data/fnm/scripts/__init__.py` | Package init |
| `data/fnm/scripts/manifest_io.py` | Manifest I/O module with typed data structures and all API functions |
| `data/fnm/scripts/tests/__init__.py` | Test package init |
| `data/fnm/scripts/tests/test_manifest.py` | 16 unit tests covering directory structure and manifest API |

## Test Results

All 16 tests passed:
- Tests 1-8: Directory/file existence and content validation against actual worktree paths
- Tests 9-16: API tests using tmp_path for isolation (roundtrip, error handling, validation, checksums)

## Design Decisions

- **FNM_DIR resolution in tests**: Uses `Path(__file__).resolve().parent.parent.parent` to find the `data/fnm/` root relative to the test file, making tests work from any checkout location.
- **Frozen dataclasses**: All data structures use `@dataclass(frozen=True)` for immutability as specified in the PRD.
- **Custom serialization**: Used explicit `_source_file_entry_to_dict` / `_dict_to_source_file_entry` helpers instead of `dataclasses.asdict` to handle `SourceFileType` enum serialization cleanly.
- **Manifest validation**: `validate_manifest_against_disk` only counts missing **required** files; optional files that are absent do not cause validation failure.

## Deviations

None. Implementation follows the PRD exactly.

## Issues

- **mypy**: Not installed in the data venv; type checking was skipped.
- **Devcontainer**: The devcontainer mounts the main repo checkout, not worktrees. Tests were run from the worktree's `data/` directory using the local `uv` and venv as specified in the PRD instructions.
