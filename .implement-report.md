# Implementation Report: PRD 03/01 — Bus Exclusion Registry

## Summary

Implemented a Python module that analyzes intermediate format bus, branch, and
transformer tables to identify all buses that must be excluded from ACPF and DCPF
verification metrics. Three exclusion categories: PSS/E-declared isolated buses
(IDE=4), de-energized buses (VM=0), and topologically disconnected buses (no
energized path to the main island containing the slack bus). Produces CSV and JSON
output files with per-bus records and aggregate summary statistics.

## Design Decisions

- Column name detection is case-insensitive with fallback to common parser variants
- IDE=4 buses are excluded from the connectivity graph before BFS runs
- VM=0 buses are NOT excluded from the connectivity graph (preserves topology)
- Three-winding transformers (K!=0) modeled as fully-connected triangle on terminal buses
- BFS component discovery with O(V+E) time complexity
- Priority ordering for multi-reason exclusions: IDE_4 > VM_ZERO > DISCONNECTED
- Uses only Python stdlib (csv, json, collections.deque, etc.) -- no external dependencies
