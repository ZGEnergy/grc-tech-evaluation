# G-FNM-3: DCPF verification against reference solution on cleaned FNM

## Result: PASS

## Approach

Loaded the pre-cleaned FNM main island case from the MATPOWER fallback path
(`data/fnm/reference/cleaned/fnm_main_island.mat`) using `loadcase()`. This
bypasses the CSV ingestion path (which failed in G-FNM-1) and loads the
network directly in MATPOWER's native format.

Solved DCPF using `rundcpf(mpc, mpopt)` with verbose output suppressed.
Compared bus voltage angles and branch active power flows against the
reference DCPF solution (`data/fnm/reference/dcpf/buses_dcpf.csv` and
`branches_dcpf.csv`).

The reference DCPF solution was generated by MATPOWER itself from the same
`.mat` file, so this test verifies round-trip consistency rather than
cross-tool accuracy. The reference CSV contains only in-service branches
(32,532), while the tool result includes all branches (32,606 including
74 out-of-service). The comparison filters tool branches to in-service
only before positional matching.

Excluded buses (2,445 isolated/disconnected buses from
`excluded_buses.json`) were removed from the bus angle metric denominator.
