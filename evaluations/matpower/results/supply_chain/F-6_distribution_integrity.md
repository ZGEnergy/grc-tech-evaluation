# F-6: Distribution Integrity

## Distribution Channel

MATPOWER is distributed as a **zip file** from GitHub Releases.

| Attribute | Value |
|-----------|-------|
| URL | `https://github.com/MATPOWER/matpower/releases/download/8.1/matpower8.1.zip` |
| Release page | `https://github.com/MATPOWER/matpower/releases/tag/8.1` |
| Asset name | `matpower8.1.zip` |
| Published | 2025-07-13 |

## Versioning

- **Versioned releases:** Each release has a Git tag (e.g., `8.1`, `8.0`, `7.1`).
- **Version in file name:** `matpower8.1.zip` includes version.
- **Version in code:** `mpver.m` returns version string at runtime.
- **Semantic-style:** MAJOR.MINOR (no patch versions).

## Integrity Verification

### Publisher-Provided Checksums

**MATPOWER does NOT publish checksums with releases.** The GitHub Release page
for v8.1 contains only the zip file — no SHA256, no GPG signature, no
SBOM (Software Bill of Materials).

### Our Verification

The project's `setup.sh` script includes a locally-computed SHA256 checksum:

```bash
MATPOWER_SHA256="7f13b1441669a64e312d14a60e564cd91977ff1676ff77d25538e94ff313dd56"
echo "${MATPOWER_SHA256}  matpower${MATPOWER_VERSION}.zip" | sha256sum -c -
```

This checksum was computed by us at download time — it is not published by
the MATPOWER project.

### Git Tag Signing

GitHub tags are not GPG-signed. The tags are lightweight (not annotated/signed).

## Alternative Distribution

- **GitHub source tarball:** Available via `https://api.github.com/repos/MATPOWER/matpower/tarball/8.1`
  but this is auto-generated by GitHub and differs from the release zip.
- **No package manager:** MATPOWER is not in any package registry (PyPI, npm,
  MATLAB File Exchange listing exists but links to GitHub).
- **No Docker image:** No official container image.
- **Previous versions:** All historical releases remain available on GitHub.

## Assessment

Distribution is straightforward (single zip from GitHub Releases) and versioned,
but lacks cryptographic integrity verification:

| Check | Status |
|-------|--------|
| Versioned releases | YES |
| Stable download URL | YES |
| Publisher checksums | NO |
| GPG signatures | NO |
| SBOM | NO |
| Reproducible builds | N/A (interpreted code) |
| Historical versions available | YES |

The absence of publisher-provided checksums is a minor supply chain gap. In
practice, GitHub's HTTPS delivery and Git tag integrity provide reasonable
assurance, but a determined supply chain attack could replace the release
artifact without detection by automated tools.
