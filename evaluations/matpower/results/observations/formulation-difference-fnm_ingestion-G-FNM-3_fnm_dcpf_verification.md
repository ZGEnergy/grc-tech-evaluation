# Observation: MATPOWER DCPF is the reference implementation -- zero formulation difference

## Finding

MATPOWER's DCPF solution is the reference against which all other tools
are compared for G-FNM-3. Since the reference DCPF solution was generated
by MATPOWER itself from the same `.mat` file, the formulation difference
decision procedure is not applicable -- MATPOWER trivially matches its own
reference at float64 machine precision.

## Context

The DCPF reference files (`buses_dcpf.csv`, `branches_dcpf.csv`) were
generated by running MATPOWER's `rundcpf()` on the cleaned FNM network.
MATPOWER uses a full B-matrix construction via `makeBdc()` that
incorporates transformer tap ratios and phase shift angles. Other tools
using simplified B-matrix construction (e.g., `b = -1/x` ignoring taps)
will show systematic deviations on branches connected to off-nominal-tap
transformers.

Max VA deviation: 5.0e-9 degrees (float64 noise).
Max branch flow deviation: 5.0e-7 percent (float64 noise).

## Implications

For cross-tool comparison: When other tools show deviations from the DCPF
reference that are concentrated on branches connected to transformers with
tap != 1.0, this should be classified as a formulation sophistication
difference, not a data ingestion error. MATPOWER's full B-matrix is the
more accurate formulation. Tools with simplified B-matrices will
systematically underperform on this metric but may still produce acceptable
results within the pass condition thresholds.
