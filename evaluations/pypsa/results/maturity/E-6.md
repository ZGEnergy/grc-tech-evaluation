# E-6: CI/CD and test coverage

## Finding

PyPSA has a comprehensive CI/CD pipeline with multi-OS, multi-Python matrix testing, 84% code coverage tracked via Codecov, static type checking with mypy, and automated security scanning via CodeQL. All recent CI runs are passing.

## Evidence

**GitHub Actions workflows (active):**

| Workflow | Purpose |
|----------|---------|
| Tests | Main test suite -- matrix of OS (ubuntu, macos, windows) x Python versions x API variants |
| Test models | Validates against solver models |
| CodeQL | Security/vulnerability scanning |
| Link Checker | Documentation link validation |
| Release | Automated PyPI publishing |
| Dependabot Updates | Automated dependency updates |

**Test configuration details (from .github/workflows/test.yml):**
- **Matrix:** ubuntu-latest, macos-latest, windows-latest x all supported Python versions x (default, new_api)
- **Coverage:** pytest with `--cov=pypsa`, coverage XML report generated, uploaded to Codecov
- **Type checking:** mypy in strict mode as a separate CI job
- **Package build verification:** Uses `hynek/build-and-inspect-python-package` to validate sdist/wheel
- **Concurrency:** Cancels in-progress runs on new pushes (efficient CI resource use)
- **Scheduled runs:** Daily cron job at 05:00 UTC to catch upstream breakage

**Code coverage:** 84% (per Codecov badge at [codecov.io/gh/PyPSA/PyPSA](https://app.codecov.io/gh/PyPSA/PyPSA))

**Recent CI status (as of 2026-03-04):**

The CI/CD setup reflects engineering maturity well beyond typical academic software.
