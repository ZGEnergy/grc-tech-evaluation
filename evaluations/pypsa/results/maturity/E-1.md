# E-1: Release cadence

## Finding

PyPSA maintains a strong and accelerating release cadence. Over the last 24 months (March 2024 through March 2026), 31 releases were published to PyPI, averaging roughly 1.3 releases per month. The most recent release at time of evaluation is v1.1.2 (2026-02-23). The project reached the v1.0 milestone in October 2025, followed by 8 additional releases in the subsequent 5 months.

## Evidence

Release count (last 24 months): 31 releases

2024 releases (from March 2024): v0.27.0, v0.27.1, v0.28.0, v0.29.0, v0.30.0, v0.30.1, v0.30.2, v0.30.3, v0.31.0, v0.31.1, v0.31.2 (11 releases)

**Last release date:** 2026-02-23 (v1.1.2)

**Semver compliance:** Yes. The project follows semantic versioning. Minor versions (0.27 through 0.35) introduced new features; patch versions (.1, .2, .3) were bug fixes. The v1.0.0 release had a proper release candidate (v1.0.0rc1). Post-1.0, MAJOR.MINOR.PATCH is properly followed.

## Implications

The release cadence signals an actively maintained project with responsive bug-fix cycles and regular feature development. The v1.0 milestone demonstrates the team's commitment to API stability and long-term support. For operational use, this cadence means security patches and bug fixes arrive promptly.
