# Observation: PyPSA DCPF produces systematic 4-degree mean deviations on 27,862-bus FNM

## Finding

PyPSA's DCPF solution (`n.lpf()`) deviates from the MATPOWER reference by a mean of
3.95 degrees across all 27,862 buses of the FNM main island, with only 9.0% of
buses within the 1.0-degree tolerance. The deviations are uniformly distributed across
all voltage tiers (230+ kV, 69-230 kV, <69 kV), not concentrated at transformer-connected
buses.

## Context

G-FNM-3 loaded the pre-cleaned FNM main island via `matpowercaseframes` +
`import_from_pypower_ppc` and solved DCPF. The reference solution was generated by
MATPOWER 8.1 on the same `.m` file. Both tools use full B-matrix construction (including
tap ratios). The deviation pattern -- uniform across voltage tiers with a -1.3 degree
mean signed bias -- suggests differences in how `import_from_pypower_ppc` maps branch
impedance parameters rather than a B-matrix formulation sophistication difference.

The branch flow comparison is better: 95.0% of branches pass the 10% tolerance
(exceeding the 90% threshold), though extreme outliers (max 87,054%) on transformer
branches trigger a hard-fail condition.

## Implications

This finding indicates that the MATPOWER-to-PyPSA conversion path
(`matpowercaseframes` -> PPC dict -> `import_from_pypower_ppc`) introduces systematic
impedance parameter differences on large networks with many transformers. This affects
the Scalability dimension (large-network fidelity), the Accessibility dimension (data
import fidelity), and should be considered when comparing PyPSA against tools with
native MATPOWER import capabilities.
