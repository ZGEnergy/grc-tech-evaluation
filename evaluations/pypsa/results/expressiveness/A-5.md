# A-5: 24-hour SCUC as MILP

## Result: PASS

## Approach

Loaded IEEE 39-bus network via the standard pypower import path. Set up 24-hour snapshots
with a realistic diurnal load profile (0.59 to 1.0 of base load). Manually assigned
generator marginal costs from MATPOWER gencost data (same workaround as A-3).

PyPSA supports unit commitment natively via `committable=True` on generators. All key UC
parameters are **built-in generator attributes**:

- `min_up_time` / `min_down_time` -- integer hours
- `start_up_cost` / `shut_down_cost` -- $/start or $/stop
- `ramp_limit_up` / `ramp_limit_down` -- per-unit of p_nom per hour
- `p_min_pu` -- minimum stable generation level
- `committable` -- boolean flag to enable UC binary variables

Reserve requirements are **not built-in** and were implemented via the
`extra_functionality` callback, which provides access to the linopy model
to add custom constraints using xarray-aligned expressions.

Solved using HiGHS MILP solver via `net.optimize(solver_name="highs")`.
