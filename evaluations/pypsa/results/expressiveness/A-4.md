# A-4: Take DC OPF Dispatch, Run Full AC Power Flow

## Result: PASS

## Approach

Ran the A-3 DC OPF to obtain optimal dispatch, then set each generator's `p_set` to
the DC OPF result and ran full AC power flow via `net.pf()`. Both steps execute on
the same PyPSA `Network` object -- no model conversion or re-import needed.

PyPSA's `pf()` method performs Newton-Raphson AC power flow using the network's full
impedance data (r, x, b) imported from the MATPOWER case file. It does not require an
external NLP solver like Ipopt -- it is a built-in direct solve.

After the AC PF, voltage magnitudes, reactive power flows, and apparent power are all
available, enabling identification of voltage and thermal violations.

Key API calls:

```python
# Stage 1: DC OPF (same as A-3)
dc_status = net.optimize(solver_name="highs", solver_options={...})

# Stage 2: Set dispatch and run AC PF

```
