# A-6: Fix Commitment from SCUC, Solve SCED as LP

## Result: PASS

## Approach

Two-stage UC/ED separation:

1. **Stage 1 (SCUC):** Run 24-hour SCUC as MILP with `committable=True` on all
   generators (identical to A-5). Extract the binary commitment schedule from
   `net.generators_t.status`.

2. **Stage 2 (SCED):** Reload the network fresh. Set `committable=False` on all
   generators (removes binary variables, making it an LP). Fix commitment by setting
   time-varying `p_max_pu=0` for hours where a generator was OFF in the SCUC solution,
   and time-varying `p_min_pu` to the minimum stable generation for hours where it was ON.
   Solve the resulting LP for economic dispatch only.

The separation is clean: SCUC determines which generators are on/off, SCED determines
optimal dispatch levels given those commitment decisions.

## Separation Mechanism

PyPSA does not have a built-in "fix commitment and re-solve" API. The two-stage
workflow requires the user to:

1. Extract `generators_t.status` from the SCUC solution
2. Reload the network (or reset solver state)
3. Translate the commitment matrix into time-varying `p_max_pu` (0 for OFF hours)
   and `p_min_pu` constraints
4. Set `committable=False` and re-solve as LP

This is straightforward but not single-API-call. The commitment matrix is a standard
pandas DataFrame, and the time-varying constraints use PyPSA's documented `_t`
time-series interface. The approach is 100% public API -- no internals needed.
