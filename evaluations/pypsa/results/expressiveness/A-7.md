# A-7: N-M Contingency Sweep (Point-Outward, Pruned)

## Result: PASS

## Approach

Implemented the full escalating, pruned contingency sweep algorithm:

1. **Graph scoping:** Used `net.graph()` to build a NetworkX graph, then `nx.single_source_shortest_path_length()` to enumerate all buses within distance x=3 of bus 16. Collected all branches (lines + transformers) with both endpoints in the subgraph.

2. **N-1 sweep:** For each scoped branch, toggled `active=False`, ran `net.lpf()` (DCPF), checked network connectivity via `nx.connected_components(net.graph(include_inactive=False))`, computed load loss from islanded components, then restored `active=True`.

3. **Pruning:** Branches whose N-1 removal caused any load loss were pruned from higher-order combinations.

4. **N-2 sweep:** Enumerated all 2-combinations of surviving branches, solved DCPF for each, pruned again.

5. **N-3 sweep:** Enumerated all 3-combinations of remaining branches.

**No model reconstruction** was needed at any step. Branch status was toggled in-place
using the documented `active` attribute, and `net.lpf()` re-solved the DCPF on the
modified network. PyPSA's `graph(include_inactive=False)` respects the active flag,
enabling connectivity checks without rebuilding.

producing just 35 combinations (vs 1,330 without pruning).

## Workarounds

None required.

All three test requirements were met using documented public APIs:

- **Graph-distance scoping:** `net.graph()` returns a NetworkX `OrderedGraph` directly.
- **Efficient contingency re-solve:** In-place `active` flag toggle + `net.lpf()`.
- **Pruning and enumeration:** Standard Python `itertools.combinations` on the branch list.
