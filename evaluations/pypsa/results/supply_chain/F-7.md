# F-7: Air-gap installability

## Finding
PyPSA and all its dependencies can be installed offline from pre-downloaded wheels. The `uv.lock` file provides hash-pinned URLs for all 87 packages. However, some PyPSA features make network requests at runtime.

## Evidence

### Offline installation path
1. **uv.lock** contains exact URLs and SHA-256 hashes for every dependency wheel/sdist
2. All packages are available as pre-built wheels for Linux x86_64 (manylinux)
3. `uv sync` with `--offline` flag can install from a local cache or vendored wheel directory
4. No build-from-source required -- all compiled extensions ship as binary wheels

### Runtime network access

The `pypsa.examples` module fetches example networks from GitHub at runtime:

```python
def _repo_url(...) -> str:
    return f"{url}v{__version_base__}/"
```

Functions like `pypsa.examples.ac_dc_meshed()` download `.nc` files from `https://github.com/PyPSA/PyPSA/raw/v1.1.2/examples/networks/...`.

The `carbon_management()` example fetches from `https://tubcloud.tu-berlin.de/`.

**These are optional convenience functions, not required for core functionality.** Users can construct networks programmatically or load from local files without any network access.

### Core functionality: no network required
- `pypsa.Network()` construction: local only
- `net.lpf()` / `net.pf()`: local only (scipy sparse solve)
- `net.optimize()`: local only (linopy + HiGHS)
- `net.export_to_netcdf()` / `net.import_from_netcdf()`: local file I/O

### Google Cloud dependencies
The `google-cloud-storage` package is installed as a transitive dependency (via dask/fsspec). It is not used by PyPSA core and does not make network calls unless explicitly configured by the user.

## Implications
Core PyPSA functionality is fully air-gappable. The only network access is the optional examples module, which can be pre-cached or bypassed by loading local network files. For ZGE deployment, the `uv.lock` with hashes provides a verified offline installation path.
