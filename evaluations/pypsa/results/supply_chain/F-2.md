# F-2: Dependency tree

## Finding
PyPSA v1.1.2 has 17 direct (non-optional) dependencies and 87 total installed packages (including transitive dependencies and dev/test tools). The dependency tree is moderately deep due to the scientific Python stack and Google Cloud libraries (transitive via linopy/xarray).

## Evidence
**Direct dependencies (17):**
numpy, scipy, pandas (>=2.0), xarray, netcdf4 (!=1.7.4), linopy (>=0.6.1), matplotlib, plotly, pydeck (>=0.6), seaborn, geopandas (>=0.9), shapely, networkx (>=2), deprecation, validators, highspy, levenshtein (>=0.27.1)

**Total installed packages:** 87

**Notable transitive dependency chains:**
- `linopy` -> `xarray` -> `dask` -> `fsspec`, `cloudpickle`, `toolz`, `partd`, `locket`
- `geopandas` -> `pyproj`, `pyogrio`, `shapely`
- `matplotlib` -> `contourpy`, `cycler`, `fonttools`, `kiwisolver`, `pillow`
- `pandera` -> `pydantic` -> `pydantic_core`, `typing-extensions`
- `google-cloud-storage` (transitive) -> `google-auth`, `google-api-core`, `protobuf`, `googleapis-common-protos`, etc.

**Depth:** Maximum dependency chain depth is approximately 5 levels (e.g., pypsa -> linopy -> xarray -> dask -> partd -> locket).

**Lock file:** `uv.lock` with pinned versions and SHA-256 hashes for all packages, ensuring reproducible installs.

## Implications
87 packages is a moderate footprint for a scientific Python tool. The Google Cloud dependencies are somewhat surprising for a power systems library but come through the broader ecosystem. The dependency count is manageable for auditing and air-gap deployment.
