# F-8: Solver dependency

## Finding
PyPSA's power flow methods (LPF, full Newton-Raphson PF) do not require any external solver -- they use scipy's built-in sparse linear algebra. PyPSA's optimization methods (OPF, LOPF) work with the open-source HiGHS solver, which is bundled as a direct dependency via the `highspy` package.

## Evidence

### Linear Power Flow (LPF) -- no solver needed

The `SubNetwork.lpf()` method solves `B * v = p` using:

```python
from scipy.sparse.linalg import spsolve
v_diff[:, 1:] = spsolve(self.B[1:, 1:], p[:, 1:].T).T
```

This is a direct sparse linear solve (SuperLU), not an optimization problem. No external solver binary is required.

### Full Power Flow (Newton-Raphson) -- no solver needed
Uses scipy's sparse LU factorization internally. Pure numerical linear algebra.

### Optimization (OPF/LOPF) -- HiGHS bundled
PyPSA delegates optimization to `linopy`, which supports multiple solvers:
- **HiGHS** (MIT license) -- bundled via `highspy` direct dependency, confirmed available
- **GLPK** -- supported, requires system install
- **SCIP** -- supported, requires system install
- **CBC** -- supported, requires system install
- **Gurobi** -- supported (optional extra `gurobipy`), commercial
- **CPLEX** -- supported, commercial
- **MOSEK** -- supported, commercial

```python
>>> from linopy.solvers import available_solvers
>>> available_solvers
['highs']
```

HiGHS is the default solver and is sufficient for all LP and MIP problems PyPSA formulates. No commercial solver license is needed.

### Test case verification
All evaluation test cases in this repo use `net.lpf()` (DCPF), which requires only scipy. For optimization tests, HiGHS is pre-installed and confirmed working.

## Implications
No commercial solver dependency. The DCPF use case requires only scipy (standard scientific Python). The OPF use case ships with HiGHS (MIT-licensed) as a bundled default. This is the strongest possible solver story for open-source deployment.
