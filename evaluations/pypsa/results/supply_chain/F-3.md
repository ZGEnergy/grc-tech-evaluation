# F-3: Dependency license audit

## Finding
All 87 installed packages use OSI-approved open-source licenses. One direct dependency (`Levenshtein`) uses GPL-2.0-or-later, which is a strong copyleft license. All other dependencies use permissive licenses (MIT, BSD, Apache-2.0, PSF-2.0, MPL-2.0).

## Evidence

### GPL-flagged dependency (DIRECT)

| Package | Version | License | Usage in PyPSA |
|---------|---------|---------|----------------|
| Levenshtein | 0.27.3 | GPL-2.0-or-later | Fuzzy matching of attribute names in `pypsa/network/transform.py` |

The `Levenshtein` package is imported directly: `from Levenshtein import distance`. It is used only for edit-distance computation to suggest corrections when users mistype attribute names. This is a UX convenience feature, not core functionality.

### License summary for all 87 packages

| License | Count | Packages (representative) |
|---------|-------|--------------------------|
| MIT | 35 | pypsa, linopy, highspy, pydantic, plotly, validators, narwhals, pytz |
| BSD-3-Clause | 20 | numpy, scipy, pandas, networkx, dask, click, fsspec, idna, toolz |
| BSD-2-Clause | 5 | Pygments, pyasn1, locket |
| Apache-2.0 | 15 | requests, xarray, google-auth, google-api-core, cryptography, rsa |
| PSF-2.0 | 1 | typing_extensions |
| MPL-2.0 | 2 | tqdm (MPL-2.0 AND MIT), certifi |
| MIT-CMU | 1 | pillow |
| GPL-2.0-or-later | 1 | Levenshtein |
| Dual BSD/Apache | 2 | python-dateutil, packaging |

### No proprietary or unknown licenses found
All packages declare recognized OSI-approved licenses via `License-Expression` metadata or PyPI classifiers.

## Implications
The GPL-2.0-or-later license on `Levenshtein` is the only concern. Under strict interpretation, linking GPL code into a proprietary application may impose copyleft obligations on the combined work. However:
1. `Levenshtein` is used only for UX string-matching, not core power flow computation
2. The `RapidFuzz` package (MIT-licensed, already installed as a transitive dep) provides equivalent functionality
3. PyPSA could be patched to use `rapidfuzz.distance.Levenshtein` instead, eliminating the GPL dependency entirely

**Severity: Low.** The GPL dependency is isolatable and replaceable. It does not contaminate the core computation path. For maximum license hygiene, ZGE could vendor a patched pypsa that swaps `Levenshtein` for `RapidFuzz`.
