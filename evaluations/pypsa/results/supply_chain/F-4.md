# F-4: Compiled extensions

## Finding
PyPSA itself is pure Python with zero compiled extensions (.so/.pyd files). Compiled extensions exist only in dependencies, all of which are well-known scientific Python packages with open-source build systems.

## Evidence

**PyPSA package:** No .so or .pyd files found in `.venv/lib/python3.12/site-packages/pypsa/`.

Dependencies with compiled extensions (248 .so files total across 28 packages):

| Package | Extension Type | Source Availability |
|---------|---------------|-------------------|
| numpy | C/Cython (core array ops, linalg, random) | Open (BSD), builds from source |
| scipy | C/Fortran (BLAS, LAPACK, sparse solvers) | Open (BSD), bundles OpenBLAS |
| pandas | Cython (indexing, parsing) | Open (BSD) |
| matplotlib | C++ (Agg renderer, path ops, ft2font) | Open (PSF) |
| shapely | Cython wrapping GEOS | Open (BSD) |
| pyproj | Cython wrapping PROJ | Open (MIT) |
| highspy | C++ (HiGHS solver core) | Open (MIT) |
| contourpy | C++ | Open (BSD) |
| kiwisolver | C++ | Open (BSD) |
| cryptography | Rust (via OpenSSL bindings) | Open (Apache/BSD) |
| pydantic_core | Rust | Open (MIT) |
| polars/polars-runtime-32 | Rust | Open (MIT) |
| pyogrio | Cython wrapping GDAL | Open (MIT) |
| netCDF4 | Cython wrapping libnetcdf | Open (MIT) |
| cffi | C (foreign function interface) | Open (MIT) |
| rapidfuzz/Levenshtein | C++ | Open (MIT/GPL) |
| bottleneck | C | Open (BSD) |
| numexpr | C | Open (MIT) |
| google-crc32c | C | Open (Apache) |

**Key observation:** The compiled extensions in the critical path for DCPF are:
- `scipy.sparse.linalg.spsolve` (Fortran/C via SuperLU) -- used directly by `SubNetwork.lpf()`
- `numpy` core operations
- `highspy` (HiGHS solver, used only for OPF, not LPF)

All are from packages with reproducible build systems and open-source toolchains.

## Implications
No opaque compiled code in PyPSA itself. All compiled extensions are from well-audited, widely-used scientific packages. The extensions can be built from source if needed for supply chain verification, though binary wheels are the standard distribution path.
