# C-6: 50-Scenario Stochastic DCPF at Scale (SMALL -- 2,000-bus network)

## Summary

PyPSA successfully solved 50 stochastic DCPF scenarios on the 2,000-bus
ACTIVSg2000 network. All 50 scenarios converged, with an average solve time of
0.91 seconds per scenario.

| Network | ACTIVSg2000 (2,000 buses) |
| Scenarios | 50 |

| Per-scenario max | 0.985 s |
| Per-scenario std dev | 0.048 s |

## Load Perturbation

Each scenario applies uniform random +/-20% perturbation to all loads
(RNG seed=42 for reproducibility).

| Metric | Value |
|--------|-------|
| Total generation range | [66,386, 67,687] MW |
| Max line flow range | [2,400, 2,471] MW |

## Method

- Base network loaded once; loads perturbed per scenario
- Each scenario solved with `net.lpf()` (in-place, no network reconstruction)
- Load values reset between scenarios

## Observations

- Per-scenario solve time is remarkably consistent (std dev = 0.048s), indicating
  that the sparse LU factorization cost is dominated by matrix structure (which
  doesn't change between scenarios) rather than numerical values.
- Memory footprint stays low (377 MB) since the same network object is reused
  across all 50 scenarios.
- The total time of 45s for 50 scenarios is well within practical limits for
  Monte Carlo studies on 2,000-bus networks.
- PyPSA's in-place load modification (`net.loads["p_set"] = ...`) makes
  scenario sweeps straightforward without requiring network reconstruction.
