# C-4: SCUC 24-hour at Scale (SMALL -- 2,000-bus network)

## Summary

PyPSA failed to solve the 24-hour Security-Constrained Unit Commitment (SCUC)
problem on the 2,000-bus ACTIVSg2000 network within the 600-second time limit.
The HiGHS MIP solver could not find a feasible solution, remaining stuck at the
root LP relaxation.

## Results

| Metric | Value |
|--------|-------|
| Network | ACTIVSg2000 (2,000 buses) |
| Generators | 544 |
| Time horizon | 24 hours |
| Binary variables | 39,168 (544 generators x 24 hours x ~3 statuses) |
| Total variables | 129,168 |
| Total constraints | 372,650 |
| Solver | HiGHS (MILP, 4 threads) |
| Wall clock | 641.4 s |
| Peak memory | 4,035 MB |
| Feasible solution found | No |
| MIP gap | infinity (no incumbent) |

## MIP Problem Size

After presolve: 134,558 rows, 85,686 cols (37,208 binary), 1,067,725 nonzeros.
The solver completed only the root LP relaxation in 600s and found no integer-feasible
solution.

## Method

- 24-hour snapshots with hour-varying load profile (0.59x--1.0x multiplier)
- UC parameters synthetically assigned by generator size: baseload (min_up=4, ramp=0.6),
  mid-merit (min_up=2, ramp=0.8), peaker (min_up=1, ramp=1.0)
- Generator costs from MATPOWER gencost (linearized quadratic)
- Solved with `net.optimize(solver_name="highs", solver_options={mip_rel_gap: 0.01})`

## Workarounds

1. **gencost not imported**: Manual cost assignment required.
2. **UC parameters not in MATPOWER**: Synthetic min_up/down_time, startup/shutdown costs,
   ramp limits, p_min_pu assigned based on generator size classification.

## Observations

- The 2,000-bus, 544-generator, 24-hour SCUC creates a MIP with ~39k binary variables,
  which is challenging for the open-source HiGHS solver.
- The root LP relaxation alone is computationally expensive (134k rows after presolve),
  preventing the solver from beginning branch-and-bound within the time limit.
- This represents a practical scalability limit for PyPSA + HiGHS on unit commitment
  problems. Production SCUC at this scale typically requires commercial solvers
  (Gurobi, CPLEX) or problem decomposition (Benders, Lagrangian relaxation).
- PyPSA's formulation includes ramping constraints, min up/down time, and
  startup/shutdown cost tracking, which significantly increase the MIP complexity
  compared to a simple economic dispatch.
