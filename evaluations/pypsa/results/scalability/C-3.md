# C-3: DC OPF at Scale with Multiple Solvers (MEDIUM -- 10,000-bus network)

## Summary

PyPSA solved DC OPF on the 10,000-bus ACTIVSg10k network with both HiGHS and GLPK
solvers. Both converged to the same optimal objective value (1,305,163.09), with
objective difference below 1e-7%.

## Results

| Solver | Wall Clock (s) | Objective | Status |
|--------|---------------|-----------|--------|
| HiGHS  | 207.8         | 1,305,163.092 | optimal |
| GLPK   | 220.0         | 1,305,163.092 | optimal |

| Metric | Value |
|--------|-------|
| Total wall clock | 428.1 s |
| Peak memory | 7,483 MB |
| Bus count | 10,000 |
| Generator count | 2,485 |
| Line count | 9,726 |

## LP Problem Size

- 43,089 rows (constraints)
- 15,191 columns (variables)
- 332,009 nonzeros

After HiGHS presolve: 4,789 rows, 6,475 cols. Solved in 6,037 simplex iterations.
GLPK required 14,645 simplex iterations.

## Method

- Zero-rated branches (rateA=0 in MATPOWER) set to `overwrite_zero_s_nom=9999.0`
  to represent unlimited capacity (the default True sets to 1.0 MW, which is
  too restrictive and causes infeasibility).
- Generator costs manually assigned from MATPOWER gencost data (linearized
  quadratic costs around initial operating point).
- Solved with `net.optimize(solver_name=...)`.

## Workarounds

1. **gencost not imported**: PyPSA's pypower importer skips gencost data. Manual
   assignment from `CaseFrames.gencost` is required.
2. **Zero-rated branches**: Must use `overwrite_zero_s_nom=9999.0` (not `True`)
   to avoid infeasibility from 1 MW branch limits on 2,459 branches.

## Observations

- HiGHS is slightly faster than GLPK (208s vs 220s) on this problem, primarily
  due to more efficient presolve and simplex implementation.
- The bulk of the wall clock time is spent on model construction by linopy
  (writing constraints/variables) and deep copy of the network between solver runs,
  not just solving.
- Both solvers produce numerically identical results, confirming the LP formulation
  is solver-independent (as expected from linopy's abstraction layer).
