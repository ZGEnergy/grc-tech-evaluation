# C-7: Solver Swap -- DC OPF (MEDIUM) with HiGHS, GLPK, SCIP

## Summary

PyPSA solved DC OPF on the 10,000-bus ACTIVSg10k network with three different
solvers: HiGHS, GLPK, and SCIP. All three converged to the same optimal objective
value. Swapping solvers requires only changing the `solver_name` parameter --
no reformulation, model changes, or code restructuring needed.

## Swap Mechanism

```python
# Solver swap is a single parameter change:
net.optimize(solver_name="highs", solver_options={...})
net.optimize(solver_name="glpk", solver_options={...})
net.optimize(solver_name="scip", solver_options={...})
```

| Property | Value |
|----------|-------|
| Reformulation needed | No |
| Code changes needed | No |
| Swap method | `solver_name` parameter only |
| Model abstraction | linopy builds LP once, exports to each solver |

## Solver Integration Details

- **HiGHS**: Direct API via `highspy`. Fastest presolve, dual simplex.
- **GLPK**: File-based interface (writes LP to temp file, reads SOL file back).
  Requires `glpsol` binary on PATH.
- **SCIP**: Via `pyscipopt` Python package. Direct API. Requires SCIP binary
  with library dependencies (libbliss, libipopt, etc.).

## Observations

- PyPSA/linopy provides a clean solver abstraction: the LP formulation is built once
  and then exported to each solver through its native interface.
- All three solvers produce numerically identical results (< 1.4e-9% difference),
  confirming the formulation is truly solver-independent.
- SCIP was slightly fastest (254s) despite being primarily a MIP solver, followed
  by HiGHS (267s) and GLPK (292s). The bulk of time is in model construction and
  deep copy, not solving (HiGHS solves the LP in 2.4s).
- GLPK and SCIP required additional system setup (Julia artifacts for binaries,
  `pyscipopt` pip package) while HiGHS works out of the box with PyPSA.
