# D-3: Example verification

## Finding

All tested getting-started examples run without modification on PyPSA 1.1.2. The
`pypsa.examples` module provides six downloadable reference networks, all of which
load successfully. The quickstart code from the documentation (build a 3-bus network
manually and run power flow) also works as written.

**6 of 6 examples tested: all pass.**

## Evidence

Tests run in devcontainer via `evaluations/pypsa/tests/accessibility/d3_example_test.py`.

| Example | Source | Result | Notes |
|---------|--------|--------|-------|
| `pypsa.examples.ac_dc_meshed()` | Built-in example loader | Pass | 9 buses, 6 generators. LPF converges. |
| `pypsa.examples.storage_hvdc()` | Built-in example loader | Pass | 6 buses, 12 generators. LPF converges. |
| `pypsa.examples.scigrid_de()` | Built-in example loader | Pass | 585 buses, 1423 generators. LPF converges. |
| `pypsa.examples.model_energy()` | Built-in example loader | Pass | 2 buses, 2920 snapshots. Loads successfully. |
| `pypsa.examples.stochastic_network()` | Built-in example loader | Pass | 3 buses, 3 scenarios, 2920 snapshots. |
| Quickstart (manual 3-bus network) | Docs (v0.33.2 quick-start page) | Pass | 3 buses, 3 lines. PF (Newton-Raphson) converges. |

### Execution details

- Examples that download from GitHub (`pypsa.examples.*`) fetch `.nc` files from the
  version-tagged release URL (`https://github.com/PyPSA/PyPSA/raw/v1.1.2/...`).
- All examples loaded and basic operations (LPF or PF) completed without errors.
- No deprecated API warnings or version-mismatch issues encountered.

### What was NOT tested

The 30+ notebook examples in the documentation gallery (e.g., unit-commitment, SCLOPF,
sector-coupling) were not individually executed. These are rendered Jupyter notebooks
that would require downloading and running each one. The `pypsa.examples` module and
quickstart code represent the "getting started" entry points most relevant to D-3.

## Implications

PyPSA's example infrastructure is well-maintained for the current release. The
version-tagged download URLs ensure examples match the installed version. This is a
strong positive signal for onboarding — a new user can load a working network and run
a solve within minutes of installing the package, provided they use the built-in
examples rather than trying to import MATPOWER `.m` files (see D-1).
