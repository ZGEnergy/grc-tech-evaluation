# D-2: Documentation audit

## Finding

PyPSA's official documentation (docs.pypsa.org) covers most Suite A test scenarios with
varying levels of completeness. Tests A-1, A-2, A-5, and A-8 are implementable from
docs alone. Tests A-3, A-4, A-6, and A-7 require supplementary source reading, GitHub
issue searching, or trial-and-error to complete.

## Evidence

### Per-test documentation assessment

| Test | Implementable from docs alone? | Notes |
|------|-------------------------------|-------|
| A-1 (DCPF) | Yes | `n.lpf()` documented in user guide Power Flow section. Linear Power Flow page covers the method. |
| A-2 (ACPF) | Yes | `n.pf()` documented with Newton-Raphson explanation, distributed slack option, and examples. Minimal example page demonstrates ACPF. |
| A-3 (DC OPF) | No | `n.optimize()` is documented for OPF. However, docs do not document that `import_from_pypower_ppc()` silently drops gencost data. A user following docs would call optimize after import and get a `ValueError: Objective function could not be created. Please make sure the components have assigned costs.` The docs do not explain how to manually map MATPOWER gencost columns to `marginal_cost`. The user must read source or search issues. |
| A-4 (AC feasibility) | Partially | The workflow of "optimize then pf" is mentioned in the power flow docs ("a common workflow is to first optimise the network...and then to run the non-linear power flow calculation"). However, extracting voltage and thermal violations from results requires knowledge of DataFrame column names not fully documented. |
| A-5 (SCUC) | Yes | Unit commitment example page covers min up/down time, startup/shutdown costs, ramp rates, and rolling horizon. The `committable` parameter and associated constraint types are documented. |
| A-6 (SCED) | Partially | No dedicated "economic dispatch" page. A user could piece together A-6 from the unit commitment docs (fix commitment, solve dispatch), but the two-stage workflow is not explicitly documented as a pattern. |
| A-7 (Contingency sweep) | No | There is a "Contingencies" user guide page and a "Security-Constrained LOPF" example, but the pruned N-M escalating sweep algorithm requires graph-distance enumeration and branch removal/restoration, which are not documented as a workflow. The user must read source to understand how to modify branch status programmatically. |
| A-8 (Stochastic) | Yes | Dedicated "Stochastic Optimization" user guide section and example. PyPSA 1.0+ added native scenario-indexed optimization. The `stochastic_network()` example demonstrates the API. |

### Documentation strengths

- Comprehensive example gallery (30+ notebook examples covering operational, planning, and sector-coupling problems)
- User guide explains mathematical formulations behind constraints
- API reference is auto-generated from docstrings
- `pypsa.examples` module provides downloadable reference networks

### Documentation gaps

- MATPOWER/pypower import limitations not documented (gencost silently dropped)
- No explicit "DC OPF from MATPOWER data" tutorial
- Import/export page lists methods but does not document data loss on import
- No documentation of the `import_from_pandapower_net` bug on multi-generator buses

## Implications

A user experienced in power systems but new to PyPSA can implement roughly half the
Suite A tests from documentation alone. The primary gap is around data ingestion from
MATPOWER format and the silent data loss in the pypower importer. For users who build
networks programmatically (the primary PyPSA workflow), documentation coverage is
substantially better.
