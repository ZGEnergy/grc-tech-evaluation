# B-3: N-1 DCPF contingency loop

## Result: QUALIFIED PASS

## Approach

The test runs N-1 DCPF contingencies on the IEEE 39-bus network (46 branches) using a
loop that clones the base network, deactivates one branch per iteration, and solves DCPF.

**Primary method (used):** `base_net.copy()` + set `active=False` + `net.lpf()`

The `copy()` method is a documented public API that deep-copies the in-memory Network
object. No re-parsing from file occurs in the loop. Each iteration:

1. `net_copy = base_net.copy()` — clone from memory
2. `net_copy.lines.at[name, 'active'] = False` — deactivate branch
3. `net_copy.lpf()` — solve DCPF (builds B/H matrices, direct sparse solve)

**Built-in method (broken):** PyPSA also provides `Network.lpf_contingency()` which
uses BODF (Branch Outage Distribution Factor) matrices for O(1)-per-contingency
computation. However, it has a bug in v1.1.2 where `pd.concat` produces a DataFrame
instead of a Series, causing `.to_frame()` to fail.

## Workarounds

Two bugs in PyPSA 1.1.2 affect N-1 contingency analysis:

1. **`lpf()` KeyError on island-causing contingencies:**
   - **What:** When deactivating a branch splits the network into sub-networks where one
     sub-network has no branches of a given type (e.g., no Lines), `sub_network.lpf()` crashes
     with `KeyError: 'Line'` at `flows.loc[:, c.name]` (line 1840 of `power_flow.py`).
   - **Why:** The flows DataFrame is indexed by `(component_type, branch_name)` from the
     active sub-network branches, but the code iterates over all passive branch component
     types regardless of whether they exist in the sub-network.
   - **Durability:** stable — The workaround is to catch `KeyError` for the 2 (of 46)
     island-causing contingencies. This uses only public APIs and is a well-understood
     edge case.
   - **Grade impact:** Minor — 44 of 46 contingencies solve cleanly. Island-causing
     contingencies are a known edge case in N-1 analysis.

2. **`lpf_contingency()` broken in v1.1.2:**
   - **What:** The BODF-based built-in method fails with `AttributeError: 'DataFrame'
     object has no attribute 'to_frame'` because `pd.concat` with a dict of Series
     produces a DataFrame when there is one snapshot.
   - **Why:** Bug in `power_flow.py` line 934.
   - **Durability:** stable — This is clearly a regression bug that would be fixed in a
     future release. The manual loop approach works.
   - **Grade impact:** The manual loop is slower than BODF but functionally complete.

base_net = _load_network(case_file)  # load once
base_net.lpf()                       # base case solve

for btype, bname in branch_list:
    net_copy = base_net.copy()       # clone from memory (not file)
    if btype == "Line":
        net_copy.lines.at[bname, "active"] = False
    else:
        net_copy.transformers.at[bname, "active"] = False
    try:
        net_copy.lpf()               # DCPF solve
    except KeyError:
        island_count += 1            # workaround for island bug
        continue
    max_load, worst_br = _get_max_loading(net_copy)
