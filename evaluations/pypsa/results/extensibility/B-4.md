# B-4: 50 Stochastic Scenarios, 24-hour Multi-Period DCPF

## Result: PASS

## Approach

Generated 50 stochastic load scenarios using an AR(1) correlated noise process applied
to a base 24-hour diurnal load profile. Each scenario produces a 24-element load
multiplier array with temporally correlated perturbations.

For each scenario:
1. Load a fresh PyPSA Network from the MATPOWER case file
2. Set 24-hour snapshots via `net.set_snapshots()`
3. Assign time-varying load via `net.loads_t.p_set` (DataFrame: 24 x N_loads)
4. Assign generator costs
5. Solve DC OPF via `net.optimize()`
6. Collect objective value and total generation

The scenario loop is standard Python -- PyPSA provides no built-in multi-scenario
framework, but the time-series interface (`_t` DataFrames) makes it straightforward
to programmatically set up each scenario.

## API for Time-Series

PyPSA's time-series interface is fully programmatic:

```python
# Set snapshots
snapshots = pd.date_range("2024-01-15", periods=24, freq="h")
net.set_snapshots(snapshots)

# Time-varying load: assign a DataFrame (24 rows x N_loads cols)
net.loads_t.p_set = pd.DataFrame(
    {load: base_load * scenario_profile for load in net.loads.index},
    index=snapshots,
)

# Solve -- snapshots are automatically used
net.optimize(solver_name="highs")
```

No special "scenario" API is needed. Each solve is independent, and the user controls
the loop. Results are collected via standard Python data structures.
