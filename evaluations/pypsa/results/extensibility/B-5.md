# B-5: Export DCPF Results to pandas DataFrame + CSV

## Result: PASS

## Approach

Ran DCPF (A-1 baseline) and exported results to pandas DataFrames and CSV files.

PyPSA stores **all results natively as pandas DataFrames** -- there is no conversion
step. The `_t` time-series attributes on the network object (`buses_t`, `lines_t`,
`generators_t`, etc.) are already DataFrames indexed by snapshot and component name.

## Output

**Lines of code beyond solve:**

| Step | Code | Lines |
|------|------|-------|
| Get bus angles | `bus_angles_df = net.buses_t.v_ang` | 1 |
| Get line flows | `line_flows_df = net.lines_t.p0` | 1 |
| Get bus injections | `bus_p_df = net.buses_t.p` | 1 |
| Write CSV 1 | `bus_angles_df.to_csv("bus_angles.csv")` | 1 |
| Write CSV 2 | `line_flows_df.to_csv("line_flows.csv")` | 1 |
| **Total** | | **5** |

Strictly speaking, the first 3 lines are just variable assignments to values that are
already DataFrames -- no conversion, parsing, or reshaping. Only the `.to_csv()` calls
are actual export operations (2 lines).

**CSV file sizes:**
- `bus_angles.csv`: 907 bytes (1 row x 39 buses)
- `line_flows.csv`: 799 bytes (1 row x 35 lines)

**Round-trip fidelity:** CSV files read back identically (within floating-point
precision) to the original DataFrames.

**All available DataFrames (no conversion needed):**

This is the simplest possible export story: PyPSA's output IS pandas, so there is
no conversion layer. Any pandas-compatible format (CSV, Parquet, Excel, HDF5, JSON)
is a single `.to_*()` call away.
