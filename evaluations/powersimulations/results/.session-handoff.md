### PowerSimulations.jl API Patterns Learned

1. **Power flow ≠ PowerSimulations.** Use `PowerFlows.jl` for DCPF/ACPF. PSI is for optimization (OPF, UC, ED).

2. **Time series boilerplate is mandatory.** Even single-period DCOPF requires:

   ```julia
   # Multiplier-based SingleTimeSeries (values 0.0-1.0, NOT absolute power)
   timestamps = [initial_time, initial_time + resolution]
   for gen in get_components(ThermalStandard, sys)
       ta = TimeArray(timestamps, [1.0, 1.0])
       ts = SingleTimeSeries("max_active_power", ta)
       add_time_series!(sys, gen, ts)
   end
   # Same for RenewableDispatch and PowerLoad
   transform_single_time_series!(sys, resolution, resolution)
   ```

3. **Must register ALL branch types:** Line, Transformer2W, TapTransformer

4. **Fix generators where active_power > Pmax** (common in MATPOWER data):

   ```julia
   for gen in get_components(ThermalStandard, sys)
       if get_active_power(gen) > get_active_power_limits(gen).max
           set_active_power!(gen, get_active_power_limits(gen).max)
       end
   end
   ```

5. **GLPK doesn't support quadratic costs** (case39 uses QuadraticCurve). Use HiGHS for LP/QP/MIP.

6. **MATPOWER has no UC params.** Must set ramp rates, min up/down times, Pmin manually for SCUC.

7. **Julia startup is ~15-30s per invocation.** Budget accordingly for test runs.

8. **Duals/LMPs:** Request via `NetworkModel(PTDFPowerModel; duals=[CopperPlateBalanceConstraint])`
