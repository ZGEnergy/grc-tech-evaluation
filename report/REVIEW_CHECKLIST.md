# Content Review Checklist

This checklist provides a structured editorial review for every page of the GRC Tech
Evaluation report site. It covers subjective quality checks that automated validation
(`make validate`) cannot assess — prose coherence, visual rendering, interactive behavior,
and navigation correctness.

## How to use

1. Run `make validate` first and fix any errors before starting this review.
2. Deploy the site locally (`make serve`) or to the staging environment.
3. Work through each page section below, checking every box as you verify it.
4. Complete the Summary Sign-Off section last, after all pages are reviewed.
5. Commit this file with boxes checked as evidence that editorial review occurred.
