# Expressiveness

## Overview

Expressiveness measures the breadth of power system analyses a tool can perform
natively or with minimal extension. It is the **highest priority** criterion in
the rubric (weight: 0.30), reflecting the core requirement that a selected tool
must cover the analysis types ZGE needs without rewriting fundamental
algorithms. Tests A-1 through A-11 map to this criterion, spanning DC power
flow (DCPF), AC power flow (ACPF), DC/AC optimal power flow (DCOPF/ACOPF),
security-constrained unit commitment and economic dispatch (SCUC/SCED),
contingency analysis, stochastic optimization, SCOPF, lossy DCOPF with LMP
decomposition, and distributed slack OPF.

Each tool was tested at three network scales — TINY (5-bus), SMALL (24-bus RTS),
and MEDIUM (ACTIVSg10k) — to expose both capability gaps and scale-dependent
failures. Tools that pass at TINY but fail at SMALL or MEDIUM reveal API
limitations that would surface in production use. Notable findings include
[probe-001](./probe-results#probe-001) (PyPSA ACPF false convergence at
MEDIUM), A-5 SCUC timeout patterns across multiple tools, and A-9 SCOPF as a
key discriminator between tools.

## Tool Results

<details className="eval-details" open>
<summary>

### PyPSA — <span className="grade-b-plus">B+</span> (#1)

</summary>

PyPSA demonstrates the strongest Expressiveness profile among all evaluated
tools. It passes A-1 (DCPF) and A-2 (ACPF) cleanly at TINY and MEDIUM scales,
though A-2 MEDIUM was downgraded to qualified pass after
[probe-001](./probe-results#probe-001) revealed false convergence (0 NR
iterations, 83% of buses at flat start). A-3 (DCOPF) and A-4 (ACOPF) pass at
TINY but fail at MEDIUM due to zero-impedance branch handling in the network
data. The A-9 SCOPF test is the key differentiator: PyPSA is one of only two
tools to pass SCOPF at SMALL scale via its native
`optimize_security_constrained()` method. The sole blocking failure is A-8
(stochastic optimization) — the `n.scenarios` data model exists but is not wired
into the optimizer.

| Test ID | TINY | SMALL | MEDIUM | Notes |
|---------|------|-------|--------|-------|
| A-1 | pass | — | pass | DCPF converges cleanly at all scales |
| A-2 | pass | — | pass | [probe-001](./probe-results#probe-001): false convergence on MEDIUM |
| A-3 | pass | — | **fail** | Zero-impedance branch handling ([T07](./sweep-findings#t07)) |
| A-4 | pass | — | **fail** | AC feasibility check; data issue at MEDIUM |
| A-5 | pass | **fail** | — | SCUC solver timeout at SMALL (HiGHS single-threaded) |
| A-6 | pass | **fail** | — | SCED cascaded from A-5 SCUC failure |
| A-7 | pass | — | **fail** | `lpf_contingency()` broken; manual loop impractical at MEDIUM |
| A-8 | **fail** | **fail** | — | Stochastic: documented but non-functional ([probe-002](./probe-results#probe-002)) |
| A-9 | pass | pass | — | SCOPF via `optimize_security_constrained()` — key differentiator |
| A-10 | pass | pass | — | Lossy DCOPF with piecewise linear loss approximation |
| A-11 | pass | pass | — | Distributed slack: PF inherently distributed by design |

**Related Probes:** [probe-001](./probe-results#probe-001) (ACPF false convergence), [probe-002](./probe-results#probe-002) (stochastic non-functional)

### PowerModels — <span className="grade-b-minus">B-</span> (#2)

</summary>

PowerModels provides the most mathematically rigorous OPF formulations among all
evaluated tools, with native support for multiple AC OPF relaxations (SOC, QC,
SDP) and a clean JuMP-based architecture that enables arbitrary constraint
injection. Core PF/OPF tests (A-1 through A-4) pass at TINY, and DCOPF scales
cleanly to MEDIUM. However, B- reflects significant gaps in the UC pipeline:
SCUC/SCED require 140-400 lines of user-assembled JuMP code and time out at
SMALL scale. Stochastic OPF is a blocking failure — the multi-network framework
supports temporal periods but lacks scenario indexing and probability weighting.
SCOPF and distributed slack are achievable through manual JuMP assembly at TINY
scale.

| Test ID | TINY | SMALL | MEDIUM | Notes |
|---------|------|-------|--------|-------|
| A-1 | pass | — | pass | DCPF via direct linear solve |
| A-2 | pass | — | **fail** | ACPF: NLsolve diverges at 10k-bus |
| A-3 | pass | — | pass | DCOPF with native LMP extraction |
| A-4 | pass | — | pass | AC feasibility check |
| A-5 | pass | **fail** | — | SCUC: ~140 LOC manual JuMP; HiGHS timeout at SMALL |
| A-6 | pass | **fail** | — | SCED: ~200 LOC manual two-stage; blocked by A-5 |
| A-7 | pass | — | **fail** | Contingency sweep: combinatorial explosion at MEDIUM |
| A-8 | **fail** | **fail** | — | Stochastic: blocking — no scenario indexing in multi-network |
| A-9 | pass | pass | — | SCOPF via manual JuMP assembly (~180 LOC) |
| A-10 | pass | pass | — | Lossy DCOPF via DCPLLPowerModel formulation |
| A-11 | pass | pass | — | Distributed slack via PTDF algebra (~150 LOC) |

### pandapower — <span className="grade-c-plus">C+</span> (#3)

</summary>

pandapower has a well-designed API for power flow analysis with clean pass on
A-1 (DCPF), A-2 (ACPF), A-3 (DCOPF), and A-4 (AC feasibility) at both TINY
and MEDIUM scales. The contingency sweep (A-7) passes at all scales using
efficient in-place branch switching. However, C+ reflects fundamental
architectural limitations: 6 of 11 expressiveness tests fail with blocking gaps.
There is no MILP optimization (blocking SCUC/SCED), no contingency-constrained
optimization (blocking SCOPF), no stochastic formulation, no lossy DC OPF, and
no distributed slack OPF. These are not missing features that could be added —
they reflect a design scope limited to single-period steady-state analysis.

| Test ID | TINY | SMALL | MEDIUM | Notes |
|---------|------|-------|--------|-------|
| A-1 | pass | — | pass | DCPF converges cleanly |
| A-2 | pass | — | pass | ACPF with multiple solver algorithms |
| A-3 | pass | — | pass | DCOPF via PYPOWER interior point (solver locked) |
| A-4 | pass | — | pass | Seamless DC OPF to AC PF feasibility check |
| A-5 | **fail** | **fail** | — | No MILP solver; no binary commitment variables |
| A-6 | **fail** | **fail** | — | No ramp rate constraints; blocked by A-5 |
| A-7 | pass | — | pass | N-M contingency sweep with graph-distance scoping |

| A-10 | **fail** | **fail** | — | No lossy DC OPF; no LMP decomposition |

### GridCal — <span className="grade-c-plus">C+</span> (#4)

</summary>

> **Note:** GridCal grades were reconstructed from sweep findings and

GridCal provides clean power flow and basic OPF capability with pass on A-1
through A-4 and A-7 at both TINY and MEDIUM scales. Lossy DCOPF (A-10) also

### PowerSimulations — <span className="grade-b-minus">B-</span> (#5)

</summary>

> **Note:** PowerSimulations grades were reconstructed from sweep findings and
> secondary test evidence. Primary synthesis was not conducted for this tool.

PowerSimulations is the only evaluated tool with native SCUC/SCED support via
built-in formulations for commitment variables, startup/shutdown costs, and ramp
constraints. A-5 (SCUC) and A-6 (SCED) pass at TINY, though SCUC on the 39-bus
network produces no unit cycling due to capacity-load ratio
([psi-F05](./sweep-findings#psi-f05)). SCOPF is achievable at TINY via manual
JuMP contingency constraint assembly with LODF-based filtering. However, ACPF
fails at MEDIUM, stochastic OPF is a blocking failure, lossy DCOPF LMP
decomposition is not supported, and many SMALL/MEDIUM tests were skipped due to
evaluation scope constraints. A potential unit mismatch between dispatch output
and system limits ([psi-F02](./sweep-findings#psi-f02)) affects confidence in
A-4 results.

### MATPOWER — <span className="grade-a-minus">A-</span> (Reference Only)

</summary>

:::note[Reference Implementation]
MATPOWER served as the reference implementation for validating test protocols
and network data. It is not a candidate for Phase 2 selection. Grades reflect

MATPOWER provides the broadest native expressiveness among all evaluated tools,
serving as the baseline against which other tools are compared. It passes A-1
through A-7 and A-9 through A-11 at TINY, with clean scaling to SMALL for most
tests. A-8 (stochastic optimization) passes at TINY via MOST but was skipped at
SMALL. The A- grade reflects one structural limitation: SCUC on the 39-bus
network produces all-committed schedules with no unit cycling
([matpower-F05](./sweep-findings#matpower-f05)), reducing the discriminative
value of the UC test.
