# Evaluation Results

This section presents the complete evaluation findings for six power-system modeling tools across six rubric criteria. Each tool was tested against a standardized protocol that executed six test suites (A through F) across three network scales: a 39-bus network (TINY), a 2,000-bus synthetic network (SMALL), and a 10,000-bus synthetic network (MEDIUM). Grades use a three-tier scale (Strong, Adequate, Weak) that aggregates per-test pass/fail outcomes within each criterion, weighting MEDIUM-scale results most heavily to reflect production relevance.

Six criterion sub-pages provide detailed per-tool evidence: [Expressiveness](./expressiveness), [Extensibility](./extensibility), [Scalability](./scalability), [Accessibility](./accessibility), [Maturity](./maturity), and [Supply Chain](./supply-chain). A [Head-to-Head](./head-to-head) comparison page addresses Phase 2-critical features directly. MATPOWER appears on every criterion page as a reference benchmark; it is not ranked because the customer requires inspectable source code, which precludes the compiled MATLAB runtime.

PyPSA is the only tool graded Strong on Expressiveness, the highest-priority criterion. It is also the only tool to achieve Strong across five of six criteria. The Julia tools (PowerModels.jl and PowerSimulations.jl) share Strong Extensibility grades due to the JuMP model-access pattern but diverge on Accessibility, where PowerModels.jl's more mature documentation earns Adequate versus PowerSimulations.jl's Weak. pandapower's Weak Expressiveness (no MILP optimization capability; A-5, A-9, A-10, A-12) and Weak Scalability (hardcoded solver with no swap mechanism; C-7, C-8) place it last among ranked tools despite Strong Maturity and Supply Chain grades.

## Ranking Methodology

Tools are ranked using lexicographic comparison across criteria in priority order:

1. **Supply Chain** (gate): any tool graded below Adequate is disqualified. All six tools pass this gate.
2. **Expressiveness**: the ability to formulate the target problem types (DC/AC power flow, OPF, SCUC, SCED, SCOPF, stochastic optimization).
3. **Extensibility**: programmatic access to inject custom constraints, swap solvers, and extract sensitivity factors.
4. **Scalability**: performance at 2,000-bus and 10,000-bus network scales.
5. **Accessibility**: documentation quality, installation friction, and onboarding effort.
6. **Maturity**: community health, release cadence, institutional backing, and bus factor.

The lexicographic method compares tools on the highest-priority criterion first. Ties on a given criterion are broken by the next criterion in priority order. This approach reflects the contract requirement that a tool must be able to express the target problems before other qualities become relevant.

**Final Ranking:**

| Rank | Tool | Deciding Factor |
|------|------|-----------------|
| #1 | PyPSA | Only tool with Strong Expressiveness (A-1 through A-12). |
| #2 | PowerModels.jl | Tied with PowerSimulations.jl through Scalability; Adequate Accessibility breaks the tie. |
| #3 | PowerSimulations.jl | Weak Accessibility (D-2, D-3, D-5) separates it from PowerModels.jl. |
| #4 | GridCal | Adequate Extensibility (not Strong) separates it from the Julia tools. |

## Sensitivity Analysis

To verify that the ranking is not an artifact of the chosen priority ordering, three alternative scenarios were tested. Each scenario re-orders the criteria while retaining the Supply Chain gate.

| Scenario | Description | #1 | #2 | #3 | #4 | #5 |
|----------|-------------|:--:|:--:|:--:|:--:|:--:|
| Baseline | Expressiveness, Extensibility, Scalability, Accessibility, Maturity | PyPSA | PowerModels | PowerSimulations | GridCal | pandapower |
| Scalability First | Scalability, Expressiveness, Extensibility, Accessibility, Maturity | PyPSA | PowerModels | PowerSimulations | GridCal | pandapower |
| Extensibility First | Extensibility, Expressiveness, Scalability, Accessibility, Maturity | PyPSA | PowerModels | PowerSimulations | GridCal | pandapower |
| Maturity/Accessibility Swap | Expressiveness, Extensibility, Scalability, Maturity, Accessibility | PyPSA | PowerModels | PowerSimulations | GridCal | pandapower |

The ranking is completely stable across all three alternative scenarios. PyPSA holds the top position in every case because it is the only tool with Strong Expressiveness, which dominates regardless of criterion priority ordering. The full ordering also remains unchanged because the tier gaps between adjacent-ranked tools are wide enough that no single priority reordering changes relative positions.

This stability means the recommendation does not depend on subjective weighting assumptions. Whether a decision-maker prioritizes scalability, extensibility, or maturity, the same tool order results.

## Criterion Sub-Pages

- **[Expressiveness](./expressiveness)**: Suite A tests covering DC/AC power flow, OPF variants, unit commitment, SCOPF, stochastic optimization, and multi-period storage dispatch.
- **[Extensibility](./extensibility)**: Suite B tests covering custom constraint injection, solver swapping, PTDF/LODF extraction, and programmatic model access.
- **[Scalability](./scalability)**: Suite C tests measuring wall-clock time, memory consumption, and convergence reliability across three network scales.
- **[Accessibility](./accessibility)**: Suite D tests evaluating documentation quality, installation friction, error messages, and onboarding effort.
- **[Maturity](./maturity)**: Suite E tests assessing release cadence, contributor diversity, CI/CD practices, institutional backing, and production deployment evidence.
- **[Supply Chain](./supply-chain)**: Suite F (gate) tests covering license compatibility, dependency audit, air-gap installability, and cryptographic provenance.
- **[Head-to-Head](./head-to-head)**: Direct comparison of the top tools on Phase 2-critical capabilities.
