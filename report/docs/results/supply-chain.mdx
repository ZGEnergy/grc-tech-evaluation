# Supply Chain

## Overview

Supply Chain is the **gate criterion** — every tool must achieve at least C+ to proceed to the full evaluation. The Suite F tests assess core licensing (F-1), dependency tree health (F-2), dependency license audit (F-3), compiled extension audit (F-4), code inspectability (F-5), distribution integrity (F-6), air-gap installability (F-7), solver dependency analysis (F-8), and getting-started integrity (F-9). These tests ensure that a selected tool can be deployed in ZGE's operational environment without licensing conflicts, supply chain vulnerabilities, or runtime barriers.

All 6 tools passed the C+ gate threshold. This makes Supply Chain the shortest criterion page — there are no gate failures to analyze and the primary differentiators are licensing model (particularly GridCal's MPL-2.0 copyleft) and dependency tree completeness. PyPSA, pandapower, and PowerModels lead with A grades reflecting clean permissive licenses, healthy dependency trees, and verified distribution integrity.

### GridCal — <span className="grade-b">B</span> (#5)

</summary>

> **Note:** GridCal grades were reconstructed from sweep findings and secondary test evidence. Primary synthesis was not conducted for this tool.

:::caution[MPL-2.0 License]
GridCal uses the Mozilla Public License 2.0, which requires that modifications to GridCal source files be distributed under MPL-2.0. This is a file-level copyleft — it does not affect code that merely imports GridCal, but any patches to GridCal itself must be shared.
:::

GridCal's MPL-2.0 license is the only non-permissive license among the evaluated tools. While MPL-2.0's file-level copyleft is less restrictive than GPL (it does not require derivative works to be open-sourced in their entirety), it does require that any modifications to GridCal's own source files be distributed under MPL-2.0. For ZGE's use case of importing GridCal as a library without modifying its source, this has minimal practical impact, but it requires ongoing license compliance awareness.

Air-gap installability failed (F-7) due to dependency on system-level packages that are not bundled in the pip distribution.

### MATPOWER — <span className="grade-a">A</span> (Reference Only)

</summary>

:::note[Reference Implementation]
MATPOWER served as the reference implementation for validating test protocols and network data. It is not a candidate for Phase 2 selection. Grades reflect protocol validation performance only.

MATPOWER uses the BSD-3-Clause license with no copyleft obligations. The tool is self-contained within the Octave/MATLAB runtime and has minimal external dependencies — the MIPS interior point solver is bundled directly. This simplicity makes MATPOWER one of the easiest tools to audit from a supply chain perspective, though the Octave/MATLAB runtime itself introduces a platform dependency.
